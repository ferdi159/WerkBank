# 📋 Merkzettel — WerkBank

Deine persönliche Wissensbasis zu WerkBank. Hier findest du alles, was
du wissen musst, wenn jemand fragt — und alles, was du selbst zur
Weiterentwicklung brauchst.

> Diese Dateien sind **nur lokal + im Git** — werden NICHT auf Vercel
> deployed (sind in `.vercelignore`).

## Wann nutze ich welche Datei?

| Datei | Wann hilfreich |
|---|---|
| **[01-was-ist-werkbank.md](01-was-ist-werkbank.md)** | Wenn jemand fragt „Was ist das?" — Pitch + alle Features im Überblick |
| **[02-systemuebersicht.md](02-systemuebersicht.md)** | Wenn du wissen willst wie die App technisch aufgebaut ist (Architektur, Speicher, Stores) |
| **[03-deployment-und-tester.md](03-deployment-und-tester.md)** | Wenn du eine neue Version live bringen willst, oder Tester anschreibst |
| **[04-faq-fuer-mich.md](04-faq-fuer-mich.md)** | Wenn ein Tester eine Frage stellt und du eine fertige Antwort brauchst |
| **[05-aenderungshistorie.md](05-aenderungshistorie.md)** | Wenn du wissen willst „Was wurde wann geändert?" — Chronik aller Commits |
| **[06-naechste-schritte.md](06-naechste-schritte.md)** | Wenn du planen willst was als nächstes kommt — Roadmap + Ideen |

## Spielregeln

- Bei jeder größeren Änderung in der App: **`05-aenderungshistorie.md`** ergänzen
- Bei neuem Feature: zusätzlich **`01-was-ist-werkbank.md`** und ggf. **`04-faq-fuer-mich.md`** updaten
- Wenn etwas Architektonisches sich ändert: **`02-systemuebersicht.md`**
- Wenn neue Idee aufkommt: in **`06-naechste-schritte.md`** notieren

## Dateien beim Suchen finden

Im VS Code / Editor: alle Dateien beginnen mit Zahl (00, 01, 02…) →
sie sortieren sich automatisch in der richtigen Reihenfolge.
